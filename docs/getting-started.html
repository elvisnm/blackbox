Get Started

Set up Blackbox for your role in under 5 minutes.

Quick Start for Developers

1

Initialize Project

Install Blackbox into your project. This creates the .blackbox/ folder and loads all 12 skills into .claude/commands/.

bbox init /path/to/your-project
2

Set Asana Token

Required for /scaffold, /refresh, and /send-back. Find this in Asana > My Settings > Apps > Developer Apps.

export ASANA_TOKEN="your-token"
3

Add Repo (Optional)

Links your repository to the dashboard. This is auto-detected during initialization if a git remote exists.

bbox add-repo owner/repo --branch main
4

Set Author (Optional)

Maps your local work to your GitHub profile. Auto-detected from GitHub CLI or your local git config.

bbox set author your-github-username

The .blackbox/ Folder

project-structure 
your-project/
└── .blackbox/
    ├── blueprints/
    │   ├── _template.md
    │   ├── feat/
    │   ├── fix/
    │   ├── improve/
    │   └── hotfix/
    └── prs/
        ├── _template.md
        ├── feat/
        ├── fix/
        ├── improve/
        ├── hotfix/
        └── reviews/

feat/

New functionality and features.

fix/

Bug fixes and error resolutions.

improve/

Refactors and performance.

hotfix/

Urgent production fixes.

Blueprint Anatomy

Asana-Owned

Generated instantly when running /scaffold.

  • Goal
  • Context
  • Requirements
  • UI / UX
  • Constraints

DEV-Owned

Fleshed out locally via /refine and during implementation.

  • Codebase Context
  • Implementation Plan
  • Technical Decisions
  • Validation

Blueprints carry only two links: the Asana ticket URL and the PR link. No status, no changelog — Asana handles all workflow tracking.

Setup by Role

Product Owner (PO)

  • Clone the project repository (read-only context).
  • Copy CLAUDE_PO.md template to project root as CLAUDE.md.
  • Configure Asana MCP with personal access token.
Key Point: PO's Claude reads the codebase for context but writes only to Asana. Never touches the repo.

Designer

  • Clone repository and configure Asana token (same as PO).
  • Copy CLAUDE_DESIGN.md to project root as CLAUDE.md.
  • Focus on UI components, styles, and pages in the "Where to Find Things" section.
Key Point: Designer's Claude researches existing UI patterns and references components by name.

QA

No Claude setup needed. QA works directly in Asana, testing against the acceptance criteria in the ticket.

Dashboard Setup (Optional)

dashboard-init
# 1. Install dependencies cd blackbox/dashboard && npm install
# 2. Add repos bbox add-repo owner/repo --branch main
# 3. Add GitHub token (Optional: increases API limit) bbox set token ghp_your-token
# 4. Run dashboard npm run dev → opens at localhost:5173
