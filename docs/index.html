Blackbox

A development Yeah! I still don't know how to call it. for AI-assisted teams.

The Problem

AI without alignment

  • • Product Owners use AI to write
  • • Designers use AI to spec interfaces
  • • Developers use AI to write code

Each one talks to AI on their own terms. Different prompts, different context, different assumptions. Every output carries its own version of the truth.

The Bridge

The contract between roles

  • • PO writes structured tickets grounded in the actual codebase
  • • Designer shapes the interface using the PO's ticket as the single source of truth
  • • Developer generates a Blueprint from the ticket — a recipe with all the context needed to plan and drive the implementation

The Flow

Skills to guide every step

  • • Refining
  • • Implementing
  • • Reviewing
  • • Shipping

Every step guided by skills that keep the AI aligned with what was actually agreed on.

No new tools. One source of truth, from the draft to the final pull request. Blackbox is just the structure that connects them.

Framework Architecture

Blueprint Template

The standardized Markdown structure that defines the feature, scope, technical plan, and testing strategy living alongside the code.

12 Skills

A suite of command-line tools to scaffold, refine, review, and wrap-up blueprints, connecting local context with Claude's API.

Role Templates

Specific CLAUDE.md guidelines and prompt structures optimized for POs and Designers to generate perfect inputs.

Dashboard

A lightweight web application that aggregates and indexes blueprints across all your projects for persistent knowledge retrieval.

Why try Blackbox?

What changes when your whole team works from the same structured path?

AI output you can trust

Every AI conversation starts from the same structured input. The ticket template and blueprint format eliminate guesswork — what AI generates is grounded in what was actually agreed on.

Less back-and-forth

Requirements are explicit from the start. Developers don't have to chase context across Slack threads, meetings, or vague tickets. The blueprint has everything they need.

Decisions that persist

Blueprints live in the repo alongside the code. Every technical decision, every requirement, every constraint — searchable and permanent, not lost in a chat window.

No new tools to learn

Product Owner and Designer stay in Asana. Developer stays in the editor. Blackbox doesn't replace your workflow — it structures the handoffs between roles so nothing gets lost.
